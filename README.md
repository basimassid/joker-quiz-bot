## 📁 الملفات

```
joker_quiz_bot/
├── bot.py            ← البوت الرئيسي
├── questions.py      ← بنك الأسئلة (عدّله لإضافة أسئلة)
├── requirements.txt  ← المكتبات
├── Procfile          ← أمر التشغيل:  worker: python bot.py
├── runtime.txt       ← إصدار بايثون
├── .python-version   ← إصدار بايثون (لـ Railway)
├── .gitignore
└── README.md
```

> ⚠️ **ملاحظة مهمة (تفادي مشكلة الكراش السابقة):** اسم الملف الرئيسي هو `bot.py` بالضبط،
> والـ `Procfile` يقول `python bot.py` — الاسمان **متطابقان**، فلن يحدث كراش بسبب اختلاف الاسم.
> إذا غيّرت اسم `bot.py` لأي اسم آخر، **لازم** تغيّر السطر داخل `Procfile` لنفس الاسم.

## 1️⃣ احصل على رمز البوت (Token)

1. افتح [@BotFather](https://t.me/BotFather) في تيليجرام.
2. أرسل `/newbot` واتبع الخطوات.
3. سينسخ لك رمزاً بهذا الشكل: `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxx`
4. احتفظ به — هذا هو `BOT_TOKEN`.

---

## 2️⃣ التشغيل محلياً (للتجربة)

```bash
pip install -r requirements.txt

# لينكس / ماك:
export BOT_TOKEN="ضع_الرمز_هنا"
python bot.py

# ويندوز (PowerShell):
$env:BOT_TOKEN="ضع_الرمز_هنا"
python bot.py
```

ثم افتح بوتك في تيليجرام وأرسل `/start`.

---

## 3️⃣ النشر على Railway عبر GitHub

1. أنشئ مستودعاً جديداً على GitHub وارفع كل ملفات هذا المجلد.
2. ادخل [railway.app](https://railway.app) → **New Project** → **Deploy from GitHub repo** → اختر المستودع.
3. بعد ما يبدأ المشروع، افتح تبويب **Variables** وأضف:

   | المتغيّر | القيمة |
   |---|---|
   | `BOT_TOKEN` | رمز البوت من BotFather (مطلوب) |
   | `ADMIN_CHAT_ID` | رقم حسابك في تيليجرام (اختياري — للإشعارات) |

4. Railway سيقرأ `Procfile` ويشغّل البوت كـ **worker** تلقائياً. ✅

> 💡 لمعرفة `ADMIN_CHAT_ID`: أرسل أي رسالة لبوت [@userinfobot](https://t.me/userinfobot) وسيعطيك رقمك.

---

## 4️⃣ حفظ لوحة الصدارة بشكل دائم (مهم على Railway)

نظام ملفات Railway *مؤقّت*: تُحفظ `leaderboard.json` أثناء التشغيل، لكنها **تُمسح عند كل إعادة نشر**.
لجعلها دائمة:

1. في Railway: **اضغط على الخدمة → Settings → Volumes → New Volume**.
2. اجعل مسار التركيب (Mount Path): `/data`
3. أضف متغيّراً جديداً: `LEADERBOARD_FILE = /data/leaderboard.json`

بهذا تبقى نتائج الطلاب محفوظة بشكل دائم.

## 5️⃣ إضافة أو تعديل الأسئلة

كل الأسئلة داخل `questions.py`. كل سؤال بهذا الشكل:

```python
{"q": "نص السؤال",
 "options": ["خيار 1", "خيار 2", "خيار 3", "خيار 4"],
 "correct": 0,            # رقم الخيار الصحيح (يبدأ من 0)
 "exp": "شرح قصير يظهر بعد الإجابة"}
```

⚠️ حافظ على **نفس عدد الأسئلة ونفس الترتيب** في القسمين `"ar"` و `"en"` حتى يبقى التقسيم موحّداً بين اللغتين.
حدود تيليجرام: السؤال ≤ 300 حرف، كل خيار ≤ 100 حرف، الشرح ≤ 200 حرف.

### 📝 أسئلة امتحان الميد (مقالية)

أسئلة الميد منفصلة في متغيّر `MID_EXAM` داخل `questions.py`، وكل سؤال بهذا الشكل:

```python
{"q": "نص السؤال المقالي",
 "a": "الإجابة النموذجية الكاملة (تظهر عند الضغط على زر الإظهار)"}
```

لا يوجد حد 200 حرف هنا لأن الإجابة تُرسل كرسالة عادية (الحد 4096 حرفاً). أضف ما تشاء من الأسئلة بنفس الترتيب في `"ar"` و `"en"`.
